Python的文件

发表于 2021-01-22 更新于 2021-04-23 分类于 Python 阅读次数： Valine：

文件，就是计算机中由操作系统管理的具有名字的存储区域。内置open()函数会创建一个Python文件对象，可以作为计算机上的一个文件链接。与我们目前见过的类型相比，文件对象多少有点不寻常，它们不是数字、序列、也不是映射。

文件操作

在 Python 中，如果想要操作文件，首先需要创建或者打开指定的文件，并创建一个文件对象，而这些工作可以通过内置的 open() 函数实现。

完整语法格式：

此格式中，用 [] 括起来的部分为可选参数，即可以使用也可以省略。其中，各个参数所代表的含义如下：

  • file：表示要创建的文件对象。
  • file_name：要创建或打开文件的文件名称，该名称要用引号（单引号或双引号都可以）括起来。需要注意的是，如果要打开的文件和当前执行的代码文件位于同一目录，则直接写文件名即可；否则，此参数需要指定打开文件所在的完整路径。
  • mode：可选参数，用于指定文件的打开模式。可选的打开模式如表-1 所示。如果不写，则默认以只读（r）模式打开文件。
  • buffering：可选参数，用于指定对文件做读写操作时，是否使用缓冲区。
  • encoding：手动设定打开文件时所使用的编码格式，不同平台的 encoding 参数值也不同，以 Windows 为例，其默认为 cp936（实际上就是 GBK 编码）。

推荐用法：

1、打开现有的文件

如果当前目录不存在想要打开的目标文件，此时调用open()函数将会报错。

2、创建当前并不存在的文件

可以通过传递mode参数，以写入模式打开文件的方式，来生成一个全新的文件，并在文件对象中写入内容。

3、写入和读取文件

在任何情况下，Python程序中的文件都采用字符串的形式进行读写。读取文件时会返回字符串str形式的文本，写文件的时候是将文本以字符串str形式传递给write()方法。文件对象常用的写入和读取方法有下面5个：

  • file.write(str)
  • file.writelines(list)
  • file.read([size])
  • file.readline()
  • file.readlines()

4、文件打开模式

表-1 open()函数支持的文件打开模式

模式 意义 注意事项
r 只读模式打开文件，读文件内容的指针会放在文件的开头。 文件必须存在。如果当前目录不存在想要打开的文件，解释器会报错。
r+ 打开文件后，既可以从头读取文件内容，也可以从开头向文件中写入新的内容，写入的新内容会覆盖文件中等长度的原有内容。
rb 以二进制格式、采用只读模式打开文件，读文件内容的指针位于文件的开头，一般用于非文本文件，如图片文件、音频文件等。
rb+ 以二进制格式、采用读写模式打开文件，读写文件的指针会放在文件的开头，通常针对非文本文件（如音频文件）。
w 以只写模式打开文件，若该文件存在，打开时会清空文件中原有的内容。 若文件存在，会清空其原有内容（覆盖文件）；反之，则创建新文件。
w+ 打开文件后，会对原有内容进行清空，并对该文件有读写权限。
都清空了，还怎么读取？！
wb 正整数以二进制格式、只写模式打开文件，一般用于非文本文件（如音频文件）
wb+ 以二进制格式、读写模式打开文件，一般用于非文本文件
a 以追加模式打开一个文件，对文件只有写入权限，如果文件已经存在，文件指针将放在文件的末尾（即新写入内容会位于已有内容之后）；反之，则会创建新文件。
a+ 以读写模式打开文件；如果文件存在，文件指针放在文件的末尾（新写入文件会位于已有内容之后）；反之，则创建新文件。
在这个状态下，文件指针在文件的末尾，还怎么读取文件？
ab 以二进制格式打开文件，并采用追加模式，对文件只有写权限。如果该文件已存在，文件指针位于文件末尾（新写入文件会位于已有内容之后）；反之，则创建新文件。
ab+ 以二进制模式打开文件，并采用追加模式，对文件具有读写权限，如果文件存在，则文件指针位于文件的末尾（新写入文件会位于已有内容之后）；反之，则创建新文件。
在这个状态下，文件指针在文件的末尾，还怎么读取文件？

鉴于实施爬虫进行数据分析的时候，我们一般只需要处理文本文件，所以暂时先不研究以二进制格式（b和b+）打开文件。这样的话，处理文本文件我们暂时只需要研究下面六种读写模式，尤其需要验证表-1中红色字体提问的三种带+的模式。
